# 第二章：CPU 和寄存器

> CPU 是计算机的"大脑"，而寄存器是 CPU 内部最快的存储单元。本章解释 CPU 为什么这么快，详细介绍 x86-64 的所有通用寄存器，并用生活类比帮助你建立直觉。

## 2.1 CPU 是什么，为什么这么快

**CPU**（Central Processing Unit，中央处理器）是计算机中执行指令的核心硬件。它的基本工作循环非常简单：

```
取指（Fetch）→ 解码（Decode）→ 执行（Execute）→ 写回（Write Back）
     ↑_______________________________________________|
                    （不断循环）
```

1. **取指**：从内存（或缓存）中读取下一条指令
2. **解码**：理解这条指令是做什么的
3. **执行**：真正执行这条指令（加法、内存读写、跳转……）
4. **写回**：把结果写回寄存器或内存

现代 CPU 每秒可以执行**数十亿条**指令。是什么让 CPU 这么快？

### 原因一：时钟频率（Clock Speed）

CPU 内部有一个"心跳"——时钟信号。现代 CPU 的时钟频率通常是 3~5 GHz，即每秒跳动 30 亿到 50 亿次。每次"心跳"，CPU 可以推进指令执行的一个阶段。

### 原因二：缓存（Cache）

CPU 访问主内存（RAM）需要大约 50~100 纳秒。CPU 时钟周期约为 0.3 纳秒。意味着如果每次指令都要访问 RAM，CPU 大部分时间在等待内存。

解决方案是**缓存**：

```
CPU 核心
  ↕ (0.3 ns)
L1 缓存（32-64 KB，每个核心独有）
  ↕ (1-3 ns)
L2 缓存（256 KB - 1 MB，每个核心独有）
  ↕ (5-10 ns)
L3 缓存（8-64 MB，多核共享）
  ↕ (50-100 ns)
主内存（RAM，几十 GB）
  ↕ (毫秒级)
磁盘/SSD
```

数据离 CPU 越近，访问越快。缓存本质上是容量小但速度极快的内存，把最常用的数据放在 CPU 附近。

### 原因三：寄存器（Registers）

**寄存器**是 CPU 内部的存储单元，比 L1 缓存还快——访问时间是 0 个时钟周期（因为它们就是 CPU 的一部分，不需要"访问"）。

x86-64 CPU 有十几个通用寄存器，每个 64 位（8 字节）。CPU 执行的大多数运算，都发生在寄存器之间。

## 2.2 寄存器类比：桌面上的便利贴

这个类比会帮助你建立直觉：

```
┌─────────────────────────────────────────────────────────────┐
│                        你的工作台                            │
│                                                             │
│  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐           │
│  │ rax│  │ rbx│  │ rcx│  │ rdx│  │ rsi│  │ rdi│  ← 便利贴 │
│  │ 42 │  │    │  │ 7  │  │    │  │    │  │ 3  │   （寄存器）│
│  └────┘  └────┘  └────┘  └────┘  └────┘  └────┘           │
│                                                             │
│  你正在处理的数字、地址、临时结果，都写在这些便利贴上         │
│                                                             │
└─────────────────────────────────────────────────────────────┘

    需要更多空间？打开抽屉（访问内存/栈）
    ——但开抽屉比看便利贴慢多了！
```

- **寄存器** = 桌面上的便利贴：数量少（只有十几张），但随手可得，读写极快
- **内存/栈** = 抽屉：容量大（几十 GB），但每次打开都要花时间
- **磁盘** = 仓库：容量极大，但几乎是"永久性存储"，访问极慢

CPU 的核心工作方式：**把数据从内存搬到寄存器，在寄存器里计算，再把结果写回内存。**

## 2.3 x86-64 通用寄存器

x86-64 架构有 16 个通用寄存器（General-Purpose Registers）：

| 寄存器 | 名称来历 | 传统用途（仅供参考，非强制）|
|--------|---------|--------------------------|
| `rax`  | Accumulator | 函数返回值，算术运算 |
| `rbx`  | Base        | 通用（被调用者保存） |
| `rcx`  | Counter     | 循环计数，第 4 个参数 |
| `rdx`  | Data        | 乘除法高位，第 3 个参数 |
| `rsi`  | Source Index | 字符串操作源地址，第 2 个参数 |
| `rdi`  | Dest Index  | 字符串操作目标地址，第 1 个参数 |
| `rsp`  | Stack Pointer | **栈顶指针**（有特殊含义！） |
| `rbp`  | Base Pointer  | **栈帧基址**（有特殊含义！） |
| `r8`   | Register 8  | 第 5 个参数 |
| `r9`   | Register 9  | 第 6 个参数 |
| `r10`  | Register 10 | 临时使用，syscall 第 4 参数 |
| `r11`  | Register 11 | 临时使用，syscall 时被破坏 |
| `r12`  | Register 12 | 通用（被调用者保存） |
| `r13`  | Register 13 | 通用（被调用者保存） |
| `r14`  | Register 14 | 通用（被调用者保存） |
| `r15`  | Register 15 | 通用（被调用者保存） |

> **注意**：`rsp` 和 `rbp` 虽然列为"通用"寄存器，但实际上有非常重要的专用含义，不能随意使用。我们在第三章详细讲。

## 2.4 寄存器命名规律：rax / eax / ax / al

这是新手最容易感到困惑的地方之一。

x86-64 的寄存器命名有历史原因。从 8086 处理器（16位）到 80386（32位）再到 x86-64（64位），每次扩展都保留了向下兼容性。

以 `rax` 为例：

```
63                                                    0
┌──────────────────────────────────────────────────────┐
│                        rax (64位)                    │
├───────────────────────────┬──────────────────────────┤
│                           │        eax (32位)        │
├───────────────────────────┴────────────┬─────────────┤
│                                        │   ax (16位) │
├───────────────────────────────────────┬┴──────┬──────┤
│                                       │ah(8位)│al(8位)│
└───────────────────────────────────────┴───────┴──────┘
```

- `rax`：完整的 64 位寄存器（8 字节）
- `eax`：`rax` 的低 32 位（4 字节）
- `ax`：`rax` 的低 16 位（2 字节）
- `ah`：`ax` 的高 8 位（1 字节）
- `al`：`ax` 的低 8 位（1 字节）

**重要行为**：在 x86-64 中，向 `eax` 写入数据会**自动清零** `rax` 的高 32 位。这是 Intel 为了避免旧代码潜在的安全问题而设计的规则。

```asm
mov rax, 0xFFFFFFFFFFFFFFFF  ; rax = 全部64位都是1
mov eax, 0x12345678          ; 写入eax：rax 变成 0x0000000012345678
                              ; 高32位被自动清零！
```

但向 `ax`（16位）写入不会清零高位：

```asm
mov rax, 0xFFFFFFFFFFFFFFFF  ; rax = 全1
mov ax,  0x1234              ; 只改低16位：rax = 0xFFFFFFFFFFFF1234
                              ; 高48位不变！
```

### 其他寄存器的命名模式

对于 `rbx`、`rcx`、`rdx`，规律相同：
- `rbx` → `ebx` → `bx` → `bh`/`bl`
- `rcx` → `ecx` → `cx` → `ch`/`cl`
- `rdx` → `edx` → `dx` → `dh`/`dl`

对于 `rsi`、`rdi`：
- `rsi` → `esi` → `si` → `sil`（没有 `sih`）
- `rdi` → `edi` → `di` → `dil`

对于新增的寄存器 `r8` 到 `r15`，命名规则不同：
- `r8`（64位）→ `r8d`（32位）→ `r8w`（16位）→ `r8b`（8位）

```asm
mov r8,  42     ; r8 的完整64位 = 42
mov r8d, 42     ; r8 低32位 = 42，高32位清零
mov r8w, 42     ; r8 低16位 = 42，其他位不变
mov r8b, 42     ; r8 低8位 = 42，其他位不变
```

### 为什么需要这些不同大小的别名？

有时候，你只关心一个整数的低 8 位（比如一个字符）或低 32 位（比如一个 int）。使用正确大小的寄存器名称：
1. 让代码更清晰地表达意图
2. 可能生成更短的机器码指令（节省字节）
3. 在处理混合大小数据时避免意外

## 2.5 特殊寄存器

除了通用寄存器，还有几个特殊的寄存器：

### rip：指令指针（Instruction Pointer）

`rip`（64位），也叫 **程序计数器**（PC），存储着**下一条要执行的指令的地址**。

```
内存中的代码：

地址 0x1000: push rbp      ← 当前正在执行
地址 0x1001: mov rbp, rsp
地址 0x1004: mov eax, 42
...

如果 CPU 正在执行 0x1000 的指令，rip 的值是 0x1001
（指向下一条指令）
```

你**不能直接**用 `mov` 指令读写 `rip`（在 32位模式下），但可以间接影响它：
- `jmp` 指令改变 `rip` 的值（跳转到新地址）
- `call` 指令把当前 `rip` 压栈，然后跳转
- `ret` 指令从栈上弹出地址到 `rip`

在 64 位模式下，`rip` 相对寻址非常常见：`[rip + 0x100]` 表示"当前指令地址加上 0x100 处的内存"。

### rflags：标志寄存器（Flags Register）

`rflags` 是一个特殊的 64 位寄存器，每一位（bit）都代表一个布尔状态。执行算术或比较指令后，CPU 会自动更新某些标志位。

最常用的标志：

| 标志位 | 名称 | 含义 |
|--------|------|------|
| `ZF` | Zero Flag | 最近一次运算结果为零 |
| `SF` | Sign Flag | 最近一次运算结果为负（最高位为1） |
| `CF` | Carry Flag | 最近一次运算产生了无符号进位/借位 |
| `OF` | Overflow Flag | 最近一次运算产生了有符号溢出 |
| `PF` | Parity Flag | 最近一次运算结果低8位中1的个数为偶数 |

**标志寄存器是条件跳转的基础**：

```asm
cmp rax, rbx       ; 计算 rax - rbx，更新标志位（但不保存结果）
je  label_equal    ; 如果 ZF=1（结果为零，即 rax == rbx），跳转

; 等价的 C 代码：
; if (rax == rbx) goto label_equal;
```

`cmp` 指令执行减法，但只更新标志位，不保存结果。然后 `je`（jump if equal）检查 `ZF` 标志位，决定是否跳转。

### 其他特殊寄存器（简要提及）

- **段寄存器**（`cs`, `ds`, `es`, `fs`, `gs`, `ss`）：在 64 位模式下大多数已废弃，但 `fs` 和 `gs` 仍用于线程本地存储（TLS）
- **浮点/SIMD 寄存器**（`xmm0`-`xmm15`, `ymm0`-`ymm15`, `zmm0`-`zmm31`）：用于浮点运算和 SIMD 向量运算，不在本章讨论范围内

## 2.6 寄存器读写示例

来看几个具体的汇编指令例子，理解寄存器是如何被使用的：

### 把数字放入寄存器

```asm
mov rax, 42         ; rax = 42（立即数）
mov rbx, rax        ; rbx = rax（寄存器间复制）
mov rcx, [rsp]      ; rcx = 内存地址 rsp 处的值（内存读取）
```

### 简单的算术

```asm
; C 代码：int x = 10; int y = 20; int z = x + y;
mov eax, 10         ; eax = 10
mov ebx, 20         ; ebx = 20
add eax, ebx        ; eax = eax + ebx = 30
; 结果在 eax 中（即 z = 30）
```

### 标志位的使用

```asm
; C 代码：if (a == 0) { ... }
; 假设 a 的值在 rax 中

test rax, rax       ; 计算 rax & rax，更新标志位
                    ; 如果 rax == 0，ZF = 1
jz  is_zero         ; jump if zero：如果 ZF=1 则跳转

; rax 不为零时执行这里的代码
...
jmp done

is_zero:
; rax == 0 时执行这里的代码
...

done:
```

`test rax, rax` 是个常见惯用法：`rax & rax` 等于 `rax` 本身，如果结果为零（即 `rax` 本身为零），`ZF` 被置 1。

## 小结

- **CPU** 通过取指-解码-执行-写回循环执行指令，每秒数十亿次
- **寄存器**是 CPU 内部最快的存储（0 个时钟周期访问延迟），x86-64 有 16 个通用寄存器
- 寄存器类比：桌面便利贴（少量但极快），内存类比：抽屉（容量大但较慢）
- **命名规律**：`rax`（64位）→ `eax`（32位）→ `ax`（16位）→ `ah`/`al`（8位）；写 `eax` 会清零 `rax` 高32位
- **`rip`**（指令指针）：存储下一条指令地址，控制程序流
- **`rflags`**（标志寄存器）：记录运算结果的布尔状态，是条件跳转的基础
- 前 6 个函数参数依次放在 `rdi`, `rsi`, `rdx`, `rcx`, `r8`, `r9`；返回值放在 `rax`（下一章详细讲）
